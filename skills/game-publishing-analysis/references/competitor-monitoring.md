# 竞品动态监控模板

## Overview

竞品分析不是一次性的。本文件提供持续监控竞品动态的框架和模板，确保发行决策基于最新市场情报。

---

## 监控维度

### 1. 竞品状态看板

每周更新一次，跟踪核心竞品的关键变化：

```
## 竞品看板 — Week of [YYYY-MM-DD]

| 竞品 | 评分变化 | 近期版本 | 重大活动 | 声量趋势 | 风险信号 |
|------|---------|---------|---------|---------|---------|
| [Name] | 4.2→4.1 ↓ | v2.3 (新英雄) | 周年庆 | 上升 | 无 |
| [Name] | 3.8→3.9 ↑ | v1.7 (bug修复) | 无 | 持平 | 评分触底反弹 |
| [Name] | 4.5 持平 | 无更新 | 赛季结束 | 下降 | 内容真空期 |
```

### 2. 评分变化追踪

| 竞品 | 本周评分 | 上周 | 上月 | 趋势 | 差评主因 |
|------|---------|------|------|------|---------|
| [Name] | 4.1 | 4.2 | 4.3 | ↓ | 服务器/匹配 |

**当竞品评分大幅下跌时（>0.2）**：
- 是发行机会 — 竞品用户可能在找替代品
- 检查差评原因，确认我们的产品是否能解决同类问题
- 考虑针对性的 UA 素材（如："稳定服务器，告别卡顿"）

### 3. 版本更新日历

```
## 竞品版本更新时间线

[竞品A] ──v2.0──v2.1──v2.2──v2.3──────
[竞品B] ────v1.5────v1.6────v1.7──────
[我们]  ──────────────── ↑ 我们的发布窗口 ↑
```

**错位竞争原则**：
- 避开竞品大版本更新后 2 周内发布（用户注意力被吸引）
- 瞄准竞品"内容真空期"（两个大版本之间的间隔）
- 竞品出严重 bug 时 = 获量窗口

### 4. 竞品活动日历

| 月份 | 竞品A 活动 | 竞品B 活动 | 我们的机会 |
|------|-----------|-----------|-----------|
| 4月 | 周年庆 | 赛季更新 | 避开，做自己的预热 |
| 5月 | 无 | 无 | ✅ 内容真空，可以发力 |
| 6月 | 世界杯活动 | 暑假活动 | 同步做世界杯，差异化内容 |

## 快速竞品 Brief 模板

当发现新竞品时，5分钟快速了解：

```
## Quick Brief: [游戏名]

**基本信息**
- 品类：
- 开发商：
- 上线时间：
- 当前评分：GP [X.X] / AS [X.X]

**规模**
- 预估月收入：$XX M
- 预估月下载：X M
- 主力市场：

**为什么关注它**
- [它跟我们有什么关系？直接竞品？间接竞品？抢同一批用户？]

**核心差异**
- 它有我们没有的：
- 我们有它没有的：

**威胁等级**
- [ ] 🔴 高（直接竞品，规模大）
- [ ] 🟡 中（间接竞品或规模小）
- [ ] 🟢 低（不同赛道，仅参考）
```

## 监控频率

| 监控项 | 频率 | 工具/来源 |
|--------|------|----------|
| 评分变化 | 每周 | Google Play / App Store |
| 版本更新 | 每次更新时 | App Store 更新日志 |
| 收入/下载估算 | 每月 | Sensor Tower / data.ai |
| 社区声量 | 每周 | Reddit / Twitter / TapTap |
| 活动日历 | 每月 | 竞品官方社媒 / Discord |

---

## 竞品情报 → 发行动作映射

| 竞品信号 | 我们的动作 |
|---------|-----------|
| 竞品评分大跌 | 加大 UA 投放，素材突出我们的稳定性 |
| 竞品大版本更新 | 暂缓投放，避免正面竞争 |
| 竞品内容真空期 | 加大投放+内容产出 |
| 新竞品进入赛道 | 快速 Brief + 评估威胁等级 |
| 竞品做大型活动 | 分析用户参与度，判断是否跟进 |
| 竞品停服/关服 | 回流营销瞄准其用户群 |
