# 发行生命周期策略手册

## Overview

不同发行阶段，目标不同、打法不同、资源分配不同。本文件定义五个阶段的核心目标、关键动作和成功标准。

---

## 阶段总览

```
探索期 → 测试期 → 软发布 → 正式发布 → 长线运营
 桌面调研   留存验证   放量筛选   全渠道爆发   回流+商业化
```

| 阶段 | 核心目标 | 预算量级 | 时间跨度 | 关键指标 |
|------|---------|---------|---------|---------|
| 探索期 | 搞清楚赛道 | 零/极低 | 2-4周 | 竞品数量、市场空白 |
| 测试期 | 验证留存 | 小预算 | 4-8周 | D7、R2、评分 |
| 软发布 | 渠道筛选+放量测试 | 中等 | 4-12周 | CPR、渠道ROI、规模化可行性 |
| 正式发布 | 爆发获量 | 全预算 | 2-4周（爆发窗口） | DAU峰值、下载量、收入 |
| 长线运营 | 留存+商业化+回流 | 稳定投入 | 持续 | MAU、ARPU、回流率 |

---

## 阶段一：探索期

### 目标
纯桌面调研，不花钱、不碰产品。搞清楚：这个品类能不能做？竞争格局是什么？

### 关键动作

**1. 同类竞品研究**
- 找出品类内所有"大成"竞品（年收入>$50M 或下载>3M）
- 使用 `competitive-intelligence.md` 框架
- 调研维度：下载量、收入估算、评分趋势、用户风评

**2. 用户风评分析**
- Google Play / App Store 评分分布
- Reddit / TapTap / Facebook 社区讨论
- 正面 vs 负面声量比
- 差评主题归类（服务器/平衡性/付费/匹配）

**3. 市场空白识别**
- 品类空白：该赛道在目标市场有无强力选手
- 体验空白：竞品核心体验有无缺陷
- 价格空白：付费设计是否让用户不满

**4. 初步 TA 画像**
- 核心玩家是谁？MOBA 用户？FPS 用户？
- 目标市场初判（SEA / LATAM / EA / EMEA）
- 参考 `regional-market-profiles.md` 做初步匹配

### 产出
```
- 竞品地图（至少3个大成竞品的详细 profile）
- 市场空白分析
- Go / No-Go 建议
- 如果 Go：推荐的测试市场 + TA 假设
```

### 成功标准
- 能清楚回答：这个品类的"大成"是谁？他们赢在哪里？
- 能清楚回答：我们的差异化是什么？
- 能清楚回答：推荐先测哪个市场？为什么？

---

## 阶段二：测试期

### 目标
验证产品留存，用最小预算回答"用户愿不愿意留下来"。

### 核心原则
- **留存是唯一目标** — 一切决策以留存数据为准
- 不做商业化宣传（Battle Pass 等暂不推）
- KOL 不大量使用，以小规模 KOC 验证为主
- 评分紧急预案触发线：低于 3.8 分启动预案

### 关键动作

**1. 小预算 UA 测试**
- 选 2-3 个市场做验证
- 以 Google UAC 为主渠道（最稳定）
- 预算：够跑出统计显著性即可（通常 $20K-50K/市场）

**2. 留存数据追踪**
- D1 / D2 / D7 / D30 逐日追踪
- D7 是 checkpoint — 是否放量看这个数据
- 按渠道分拆留存（Google vs TikTok vs Organic）

**3. 评分管理**
- 每日监控 Google Play / App Store 评分
- 差评主题归类，优先修 Top 3 差评主题
- 目标：维持 4.0+ 或提升到 4.0+

**4. 产品问题记录**
- 收集用户反馈（社区 + 评论 + 客服）
- 区分"外围问题"和"核心玩法问题"
- ⚠️ ingame 痛点改动 >> 外围活动/营销

### 关键指标

| 指标 | 红线 | 及格 | 优秀 |
|------|------|------|------|
| D2 (R2) | <30% | 30-40% | >40% |
| D7 (R7) | <8% | 8-15% | >15% |
| 评分 | <3.8 ⚠️ | 3.8-4.2 | >4.2 |
| Organic% | <20% | 20-40% | >40% |

### 产出
```
- 留存数据报告（按市场、按渠道）
- 渠道效率排名（CPR、CPI 对比）
- 产品问题优先级列表
- Go / No-Go 到软发布的决策
```

## 阶段三：软发布

### 目标
在验证留存的基础上，测试能否规模化获量，筛选最优渠道组合。

### 核心原则
- 留存数据仍需持续监控（不能因为放量而牺牲留存）
- 渠道筛选是核心 — 哪个渠道能以合理 CPR 带来高质量用户
- 开始小规模测试商业化（如果产品准备好了）

### 关键动作

**1. 渠道放量测试**
- Google UAC 放量（通常是测试期的 3-5x 预算）
- TikTok 测试（配合 KOL 内容，纯付费往往效果差）
- Apple Search Ads 小预算测试（高质量但贵）
- Facebook 视市场情况决定（SEA 通常效果差）

**2. 素材优化**
- A/B 测试不同创意方向
- 按市场定制本地化素材
- 参考 `ua-creative-guide.md`

**3. KOL 小规模试水**
- 选 3-5 个中小 KOL 做内容测试
- 验证哪种内容类型带来最高安装转化
- 非爆发期预算有限，以效果验证为主

**4. 市场扩展测试**
- 如果主力市场验证成功，开始测试第 2-3 个市场
- 每个新市场走"小预算验证"流程

### 关键指标

| 指标 | 关注点 |
|------|--------|
| CPR 趋势 | 放量后 CPR 是否可控（<1.5x 测试期 CPR） |
| 渠道 ROI | 各渠道的 LTV/CPA 比 |
| 留存稳定性 | 放量后 D7 是否保持 |
| 规模天花板 | 每日获量上限（Daily Cap） |

### 产出
```
- 渠道效率矩阵（渠道 × 市场 × CPR × 留存）
- 最优渠道组合建议
- 正式发布预算模型
- 正式发布时间窗口建议
```

## 阶段四：正式发布

### 目标
全渠道爆发，在最优时间窗口最大化获量和声量。

### 核心原则
- 时间窗口决定一切 — 趋势窗口 > 拍脑袋
- 全渠道协同（OM + KOL + PR + 社群同步发力）
- 爆发期通常 2-4 周，必须提前准备

### 关键动作

**1. 全渠道 OM 铺量**
- 所有验证过的渠道同步放量
- 预算分配按软发布期的效率数据决定
- 预留 20% 预算做实时调优

**2. KOL 集中爆发**
- 头部 KOL（1-3个）做声量引爆
- 中腰部 KOL（10-20个）做内容覆盖
- 发布前 1-2 周开始预热内容
- 发布日集中释放

**3. PR + 媒体**
- 游戏媒体提前 2 周发新闻稿
- 社交媒体倒计时活动
- 发布日直播/首发活动

**4. 商店优化**
- ASO 关键词更新
- 截图/视频素材更新
- 特色功能展示

**5. 社群爆发**
- Discord/Facebook 社群活动
- 首发奖励机制
- 玩家 UGC 征集

### 关键指标

| 指标 | 目标 |
|------|------|
| 发布日下载量 | 品类 Top X 水平 |
| 首周 DAU | 预设目标值 |
| 首周收入 | 商业化预期验证 |
| 首周评分 | 维持 4.0+ |
| 媒体覆盖 | 主流游戏媒体报道 |

## 阶段五：长线运营

### 目标
保持用户活跃，推进商业化，建立回流机制。

### 核心维度

**1. 回流用户策略**
- OM 定向再营销（针对注册但流失的用户）
- 回流 KOC 内容（真实回流体验 > 大 KOL 推广）
- 邮件/推送召回（版本更新、活动提醒）
- 老玩家邀请机制

**2. 商业化推进**
- Battle Pass / Season Pass 上线
- 限时活动商业化（节日皮肤、联名活动）
- 根据付费数据调优定价

**3. 社群长线运营**
- 定期社群活动（周赛、月赛）
- 玩家 UGC 生态建设
- 开发者日志/沟通机制
- 培养游戏专属 KOC 和素人内容创作者

**4. 内容更新节奏**
- 大版本更新节奏（季度/半年）
- 小版本/热修复节奏（双周/月）
- 每次大更新配合一轮回流 + 获量

### 关键指标

| 指标 | 关注频率 |
|------|---------|
| MAU / DAU | 每日 |
| ARPU / ARPPU | 每周 |
| 回流率（Reactivation Rate）| 每次活动后 |
| 付费渗透率 | 每月 |
| 社群活跃度 | 每周 |

---

## 阶段转换决策点

```
探索期 → 测试期：Go/No-Go 基于市场空白和差异化确认
测试期 → 软发布：D7 达标 + 评分 ≥ 3.8 + 至少1个市场验证
软发布 → 正式发布：CPR 可控 + 渠道组合确认 + 时间窗口选定
正式发布 → 长线运营：自然过渡，发布后 4 周进入
```
