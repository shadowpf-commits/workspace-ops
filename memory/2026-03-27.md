# 2026-03-27 Daily Notes

## WeChat Bot (clawbot) 401 Error 故障排查

### 症状
微信发消息给 bot → 回复 401 "No API key found for provider 'anthropic'" → 循环重试 → 消息队列积压 90+ → 多次错误通知轰炸微信用户

### 根本原因
1. **auth profile 错误配置**：`anthropic:default` 在 main agent 的 auth-profiles.json 里存的是 MiniMax 的 key（sk-cp-...），不是真实的 Anthropic API key。embedded agent fallback 到 `anthropic/claude-opus-4-6` 时调 Anthropic API 必然 401。
2. **session 文件锁**：PID 12715 的 openclaw-gateway 持有一个 session 文件锁不释放（`c1dcd3c2-7e37-4180-86ca-820a2ea71058.jsonl.lock`），导致所有请求排队超时，超时后又触发重试通知，恶性循环。
3. **路由问题**：binding 配置了 `agentId=wechat, channel=openclaw-weixin`，但消息始终路由到 main agent 而不是 wechat agent。原因未明（binding 配置正确，channel 名称匹配 "openclaw-weixin"，但 resolveAgentRoute 就是不返回 wechat）。

### 修复操作
- 删除了 main agent 的 `anthropic:default` profile（两个文件：auth-profiles.json 和 authprofiles.json）
- 给 wechat agent 创建了专属配置：
  - `/Users/shadow/.openclaw/agents/wechat/agent/models.json` — 仅 minimax/minimax-portal 模型
  - `/Users/shadow/.openclaw/agents/wechat/agent/auth-profiles.json` 和 `authprofiles.json` — minimax key
- 删除了残留的 session 锁文件（多次）
- 杀掉了僵尸进程：PID 9323 (openclaw), PID 9325 (openclaw-sessions, 占用 2.6GB)
- 恢复了 wechat binding
