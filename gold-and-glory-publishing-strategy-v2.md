## 0. 方案结构说明：全球主轴 × 区域适配

> ⚡ *阅读本方案前必读：本节说明整体架构，确保理解"哪些是全球统一"，"哪些是区域差异"，以及"为什么不同"。*

---

### 0.1 核心原则：一个主轴，多个区域执行

G&G 的发行策略不是"一套方案复制到所有市场"，而是：

> **一个主轴（Global Axis）× 多个区域执行（Regional Execution）**

- **主轴** = G&G 的产品身份和全球统一决策，**所有市场一致**
- **区域执行** = 在主轴框架下的执行差异，**不同市场有不同打法**

---

### 0.2 全球统一主轴（所有市场不变）

以下要素是 G&G 的核心身份，**任何市场都不改变**：

| # | 主轴要素 | 全球统一内容 | 为什么不变 |
|---|---------|------------|-----------|
| 1 | **核心定位** | "5分钟中世纪地牢Extraction RPG" | 差异化核心，改变即失去独特性 |
| 2 | **变现模型** | F2P + Battle Pass + 英雄直购 | 品类验证最优解，不因市场而改 |
| 3 | **内容节拍** | 每6–8周一赛季，每赛季出新英雄 | 运营节奏和社区期待建立在稳定节奏上 |
| 4 | **核心叙事** | Gameplay展示"5分钟→地牢→战斗→提取→爽" | 转化率最高的叙事方式，跨文化有效 |
| 5 | **赛季KPI框架** | D1 > 30%，D7 > 15%，CPA < $15 | 全球统一的健康度基准 |
| 6 | **Battle Pass结构** | 50级结构，$9.99美国基准价 | 结构统一，参数（价格/奖励）按区域调整 |
| 7 | **评分改善目标** | 3.7 → 4.0，全球同步优先级 | 评分影响ASO，所有市场受益 |
| 8 | **KOL分级框架** | Nano/Micro/Mid/Macro四级分层 | 分层逻辑通用，各市场套用此框架 |

### 0.3 区域执行差异（为什么不同——原因驱动）

下表列出各维度在哪些市场有差异，以及**为什么会不同**：

| 维度 | 差异市场 | 差异内容 | 差异原因 |
|------|---------|---------|---------|
| **定价** | 印度 / 拉美 / 东南亚 | Battle Pass 价格不同（$1.99–$9.99）| 购买力差异（购买力平价），竞争定价压力 |
| **KOL切入角度** | 每个市场都不同 | 菲律宾：社交；印度：性价比；越南：攻略 | 文化偏好和用户使用场景不同 |
| **主要投放平台** | 每个市场权重不同 | 印尼 TikTok > Meta；印度 Google UAC > TikTok | 各市场平台渗透率和使用习惯不同 |
| **素材本地化深度** | 拉美 / 越南 | 本地语言 + 本地文化二创 | 文化认同影响转化率 |
| **节日营销** | 各市场不同 | 亡灵节（墨西哥）；独立日（印尼）；排灯节（印度）| 各市场有独特文化节点，配合活动效果更好 |
| **活跃时段** | 各市场有时差 | 越南UTC+7；墨西哥UTC-6 | 投放和运营需要匹配目标用户在线时间 |
| **支付方式** | 哥伦比亚 | Nequi/PSE 支付支持 | Google Play 以外的主流支付方式 |
| **KOL层级侧重** | 各市场不同 | 印度Micro/Nano为主；墨西哥Mid为主 | 各市场KOL生态成熟度不同，报价和可及性差异 |

## 1. Game & Category Analysis

### 1.1 Category & Core Loop

| Dimension | Detail |
|-----------|--------|
| **Primary Genre** | RPG (Role Playing Game) |
| **Hybrid Genres** | Battle Royale + Extraction Shooter + Hero-based MOBA |
| **Monetization** | Free-to-play (F2P) with IAP + Seasonal Battle Pass |
| **Session Length** | ~5 minutes (dungeon raid) |
| **File Size** | ~840 MB (optimized for low-end devices) |
| **Platform** | Android (Google Play primary); iOS likely planned |
| **Developer** | Studio Dois Private Limited |

**Core Gameplay Loop:**
```
选择英雄 (Hero Selection)
    ↓
进入地牢 (Enter Dungeon)
    ↓
PVE打怪 + PVP抢宝 (Combat + Loot PvE monsters + Fight other players)
    ↓
提取生存/战败全失 (Extract with loot OR lose it all on death)
    ↓
资源强化英雄 (Upgrade hero with extracted resources)
    ↓
Ranked / Guild 循环 (Climb ranks, guild battles)
```

---

### 1.2 Unique Features

Gold and Glory occupies a specific intersection that few mobile games nail:

- **"First mobile medieval hero-based PVP dungeon exploration"** — self-description，直接点出了差异化
- **Extraction mechanic on mobile** — PC端（Escape from Tarkov, DMZ）验证过的玩法，手游端目前无强力竞品
- **5分钟快节奏** — 对比同类地牢游戏（需要30分钟+），极大降低门槛
- **Hero-based MOBA-like combat** — 每个英雄有独特技能，不是单纯数值碾压
- **轻度包体** — 840MB，适配低端机，受众更广
- **赛季制 + 新英雄更新** — 有内容持续更新的运营模式

**一句话定位：** "Quick-session medieval extraction RPG — dungeons you can finish in a coffee break."

## 2. Target Audience Deep-Dive

### 2.1 用户画像与动机分层

⚡ *以下分层基于品类规律和公开数据推断*

| 年龄段 | 核心动机 | 典型场景 | 付费逻辑 | 流失风险 |
|--------|----------|----------|----------|----------|
| **16–18岁** | 社交认同 + 朋友都在玩 | 下课后/宿舍，30分钟–2小时；和朋友语音组队 | 低，月均 $1–3；主要是时间投入；Battle Pass勉强 | 中（学业影响） |
| **18–24岁** | 竞技上分 + 展示技术 | 碎片时间多次登录；深夜时段活跃 | 中，月均 $5–15；Battle Pass主力用户 + 英雄皮肤 | 低（习惯形成期） |
| **25–30岁** | 轻松娱乐 + 怀旧中世纪题材 | 通勤/午休，1–2个完整地牢 | 高，月均 $15–30；付费意愿最强，直接买英雄 | 低 |
| **30–35岁** | 情怀 + 轻度社交 | 偶发性登录，怀旧驱动 | 低–中，月均 $3–8；倾向一次性购买 | 高（生命周期短） |

**主力用户锁定 18–28岁**，原因：
- 有竞技驱动（愿意钻研英雄强度、攻略）
- 有一定可支配收入（Battle Pass + IAP）
- 社交需求强烈（愿意和朋友组队、分享精彩片段）
- 中世纪/奇幻题材在他们成长阶段有文化积累（魔兽世界、Rift时代影响）

### 2.2 核心 Hook：为什么选 G&G 而不是其他游戏？

三个不可替代的留存锚点：

**Hook 1：「5分钟打一把地牢」**
- 问题：传统RPG地牢需要30–60分钟，手游玩家实际平均游戏时长约8–15分钟
- G&G解法：用快节奏session解决了"没时间"的焦虑
- 证据：Google Play评论中高频词包含"quick"、"fast"、"5 minutes" ⚡推断

**Hook 2：「和朋友组队抢装备」**
- 东南亚手游用户有强烈社交游戏习惯（参考 Mobile Legends 在菲律宾/印尼的爆发）
- Extraction机制天然制造"我和朋友一起冒险"的共同记忆
- TikTok传播素材丰富：组队开黑片段、抢装备的戏剧性时刻

**Hook 3：「死了装备全没，活着带出超爽」**
- 稀缺性和损失厌恶是强留存机制（Darkest Dungeon、Extract Shooter品类验证过）
- 幸存者偏差→胜利炫耀→社交传播的天然闭环
- "我刚从地牢带出一把橙装"是天然的TikTok病毒内容

## 4. Rating 专项改善方案

⚡ *基于Google Play评论高频问题规律推断*

### 4.1 当前评分问题诊断

**3.7分的根因通常集中在以下几类（按概率排序）：**

| 问题类型 | 典型表现 | 优先级 |
|----------|----------|--------|
| **新手引导不足** | 前5分钟迷茫、不知道干什么、卡关 | P0 |
| **服务器/维护** | 临时维护公告多、掉线、卡顿 | P0 |
| **匹配机制差** | 新手被老手碾压、匹配等待久 | P1 |
| **外挂/作弊** | 竞技体验被破坏、举报无响应 | P1 |
| **内容深度不足** | 玩几天就没东西玩了 | P2 |
| **付费坑太深** | 零氪体验差、Pay-to-Win感强 | P2 |

### 4.2 30天评分拯救计划

**Week 1–2：问题收集 + 快速修复**
```
□ 在游戏内增加"反馈入口"（低摩擦，截图+描述）
□ 收集Google Play最近100条1–2星评论，分类统计TOP3问题
□ 服务器稳定性专项：增加服务器巡检频率，减少临时维护
□ 新手引导优化：增加前5分钟的任务指引和语音提示
```

**Week 3–4：口碑引导 + 运营动作**
```
□ 触发评分弹窗优化：仅在用户完成第一个地牢后、情绪正向时触发
  （避免在死亡/失败后触发，会集中刷1星）
□ Discord/社群招募核心玩家，定向引导5星好评
□ 官方社区发布"我们听到了你的声音"公告（不道歉，但展示改进）
```

**Week 5–8：长期机制建立**
```
□ 建立"差评-客服"直达通道，48小时内响应
□ 反外挂专项：引入AI检测 + 人工审核
□ D1留存改善追踪：目标从当前估算<30% 提升至 35%+
□ 每赛季末做"玩家满意度调研"，纳入版本迭代参考
```

### 4.3 评分KPI目标

| 时间节点 | 目标评分 | 策略重点 |
|----------|----------|----------|

### 5.2 墨西哥测试推断分析

⚡ *基于"已上线测试"的已知事实 + 市场规律推断*

- 下载量区间：估计 50–200K（测试期合理范围）⚡推断

- 付费率：估计 1–2%（Battle Pass为主）⚡推断

**墨西哥用户洞察：**
- 中世纪/奇幻题材在墨西哥有文化接受度（受欧美游戏影响）

- 本地西班牙语主播的推荐对下载决策影响大
- TikTok视频中"和朋友一起玩"的片段传播力强

**墨西哥专项策略：**

| 维度 | 策略 |
